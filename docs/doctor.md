# 医生使用指南

Doctor 模式面向临床医生，输出证据档案式摘要，可作为临床背景参考。

> 输出内容为教材知识摘要，**不构成处方依据**，实际诊疗须结合具体患者。

---

## 启用方式

```bash
# 基本调用
./bin/ask.sh --mode doctor "高血压血压控制目标？"

# 深度核验（原子声明 grep + 回炉自纠，降低幻觉，耗时约 2×）
./bin/ask.sh --mode doctor --deep "心衰射血分数保留型（HFpEF）管理？"

# 强制指定专科（跳过自动路由）
./bin/ask.sh --mode doctor --domain renal:ckd "CKD患者血压管理循证方案？"
./bin/ask.sh --mode doctor --domain cardiology:heart_failure "HFpEF的循证管理？"
```

---

## 输出结构

| 段落 | 内容 |
|------|------|
| 【定义与流行病学】 | 核心定义 + 流行病学要点（2-4 句） |
| 【循证管理】 | 4-8 条管理建议，每条末尾标注证据等级 |
| 【红旗症状/转诊指征】 | `[症状] → [建议动作]（立即/急诊/专科）` |
| 【证据等级汇总】 | 将循证管理各类标注汇总为表格 |
| 【参考】 | 逐条列出引用来源（教材页码 + 指南名称年份） |

---

## 证据等级标注

出现在【循证管理】每条建议末尾括号内：

| 标注 | 对应依据 |
|------|---------|
| `(高级别证据)` | RCT / 系统综述 / Meta 分析 |
| `(中级别证据)` | 队列研究 / 指南弱推荐 |
| `(低级别证据)` | 专家共识 / 病例系列 |
| `(指南推荐·[指南名][年份])` | 引用具体指南原文 |

教材与指南冲突时，以最新指南为准；【参考】同时引用教材章节和指南名称/年份。

## 越界行为（doctor 模式）

与 patient 模式相同的拒答类别：

| 类别 | 仍然拒答 |
|------|---------|
| A 外科/介入决策 | 搭桥/支架/移植时机 |
| B 化疗具体方案 | 化疗药选择、剂量 |
| C 诊断红线 | 确诊/读化验单/影像解读 |
| D 调药红线 | 具体剂量调整指令 |

Doctor 模式**放宽**鉴别诊断框架讨论（可列鉴别要点），但不给出最终确诊结论。
